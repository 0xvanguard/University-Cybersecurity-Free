## 📋 Descripción

Rol **generalista** que toca múltiples áreas — alertas, vulnerabilidades, controles, evidencias, reportes. **No es lo mismo que SOC Analyst** (más reactivo a alertas) ni que Vuln Analyst (más patches). Es el "Swiss Army Knife" de seguridad en pymes y MSSPs, especialmente útil en orgs sin SOC formal.

Es **el segundo título más común** en LinkedIn de ciberseguridad. Suele ser el primer rol post-bootcamp / post-Security+. Desde aquí se especializa: hacia SOC, GRC, AppSec, IR o Cloud.

---

## 👤 Roles típicos

- **Security Analyst** — generalista en pyme / MSSP
- **Junior Cybersecurity Analyst** — primer rol post-cert
- **Information Security Analyst** — más papelero (políticas + soporte audit)
- **MSSP Analyst** — multi-cliente, multi-tarea
- **IT Security Analyst** — sysadmin con sombrero seguridad

## 🗺️ Roadmap (12 meses)

### Fase 1 — Fundamentos (0-3 meses)
- [ ] CompTIA Security+ aprobado
- [ ] Wazuh + 2 endpoints generando alertas
- [ ] OWASP Top 10 + MITRE ATT&CK leídos
- [ ] LinkedIn optimizado + 5 writeups TryHackMe en GitHub

### Fase 2 — Especialización (3-6 meses)
- [ ] Decidir vertical: SOC | GRC | AppSec | Cloud | IR
- [ ] Cert vertical correspondiente (BTL1, CySA+, SC-200, AWS Practitioner)
- [ ] LetsDefend SOC path o equivalente
- [ ] Networking: 1 evento mensual / Discord servers de ciberseguridad LATAM

### Fase 3 — Profesional (6-12 meses)
- [ ] Conseguir el primer rol especializado (transición desde generalista)
- [ ] Cert mid del vertical elegido
- [ ] Portafolio público (blog / GitHub / talks)
- [ ] Identificar mentor/a en el vertical
