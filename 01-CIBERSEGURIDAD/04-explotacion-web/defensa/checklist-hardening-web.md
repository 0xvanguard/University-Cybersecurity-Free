# 📝 Checklist de hardening web

> Lista rápida para revisar tras explotar y antes de re-testear.

---

## Entrada y lógica

- [ ] Inputs validados en servidor.
- [ ] Consultas parametrizadas.
- [ ] Controles de acceso por recurso.

---

## Headers y transporte

- [ ] Headers de seguridad aplicados.
- [ ] Uso correcto de HTTPS (cuando aplique).

---

## Logs y monitoreo

- [ ] Intentos de payloads registrados.
- [ ] Accesos sospechosos a recursos ajenos registrados.
