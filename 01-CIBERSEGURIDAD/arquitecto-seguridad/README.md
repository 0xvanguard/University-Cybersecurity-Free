## 📋 Descripción

Rol **estratégico-técnico** que define la arquitectura de seguridad de una organización: zonas, controles, integración entre sistemas, principios (Zero Trust, defense-in-depth, least privilege) y estándares. No se opera el día a día — se **diseña la cancha** donde otros juegan.

Es un destino, no un punto de partida. Llegan ingenieros senior con 5-10+ años en redes, cloud, AppSec o IAM. Demanda creciente por migraciones cloud + Zero Trust + complejidad regulatoria.

---

## 👤 Roles típicos

- **Security Architect** — generalista, multi-dominio
- **Cloud Security Architect** — AWS/Azure/GCP, landing zones, ZTNA
- **Enterprise Security Architect** — alinea con TOGAF / Zachman
- **Solution Security Architect** — específico a un producto/proyecto
- **Zero Trust Architect** *(emergente)* — diseño ZT end-to-end

## 🗺️ Roadmap (12 meses, asumiendo ya senior técnico)

### Fase 1 — Fundamentos arquitectónicos (0-3 meses)
- [ ] NIST 800-207 (Zero Trust Architecture) leído + sintetizado
- [ ] AWS Well-Architected Security Pillar completo
- [ ] Threat modeling: 3 modelos STRIDE de sistemas reales
- [ ] CCSP o TOGAF Foundation aprobado

### Fase 2 — Diseño aplicado (3-6 meses)
- [ ] Diseñar landing zone AWS (multi-account + SCPs + Control Tower)
- [ ] Diagrama Zero Trust completo para una pyme (red + identidad + endpoint)
- [ ] ADRs (Architecture Decision Records) escritos para 5 decisiones
- [ ] CISSP empezado

### Fase 3 — Liderazgo arquitectónico (6-12 meses)
- [ ] Liderar diseño de un proyecto crítico (real o simulado)
- [ ] SABSA Foundation o AWS SA Pro aprobado
- [ ] Charla técnica o paper sobre un patrón arquitectónico
- [ ] Mentorear ≥1 ingeniero hacia senior
