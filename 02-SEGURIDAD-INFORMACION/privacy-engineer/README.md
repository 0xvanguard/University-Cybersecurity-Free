## 📋 Descripción

Rol **emergente y técnico** que implementa privacidad **en código y arquitectura**. A diferencia del DPO (legal-regulatorio), el Privacy Engineer **escribe** los controles: minimización, anonimización, pseudonimización, consent management técnico, data subject request automation, retention enforcement, **PETs** (Privacy-Enhancing Technologies).

Es el **mejor pagado del lado privacy** porque combina tres skillsets escasos:
1. Ingeniería de software (idealmente backend / data)
2. Privacy law literacy (GDPR, Ley 1581, CCPA)
3. Cripto/ML aplicada (DP, FHE, MPC, federated)

Big tech (Google, Apple, Meta, Microsoft) tiene equipos dedicados de Privacy Engineering. Empresas medianas apenas lo están descubriendo en LATAM 2025-2026.

## 👤 Roles típicos

- **Privacy Engineer** — generalista
- **Privacy Software Engineer (Big Tech)** — embebido con producto
- **Privacy Architect** — diseño de sistemas privacy-by-design
- **PETs Engineer** *(escaso)* — DP, FHE, MPC, federated learning
- **Data Anonymization Specialist** — k-anonymity, l-diversity, t-closeness
- **AI Privacy Engineer** *(emergente)* — privacy en sistemas ML

## 🗺️ Roadmap (12 meses)

### Fase 1 — Fundamentos (0-3 meses)
- [ ] GDPR + Ley 1581 leídos desde lente técnico
- [ ] CIPT empezado
- [ ] Lab: anonimización de dataset con ARX (k=5, l=2)
- [ ] OpenMined Privacy ML course iniciado

### Fase 2 — PETs aplicadas (3-7 meses)
- [ ] Differential Privacy con Opacus en modelo PyTorch
- [ ] FHE básico con TenSEAL: suma + multiplicación encriptada
- [ ] Federated learning con Flower (3 clientes simulados)
- [ ] **CIPT aprobado**

### Fase 3 — Profesional (7-12 meses)
- [ ] Pipeline DSAR automatizado (Python + cloud APIs)
- [ ] Privacy review de un sistema real (open source) + recomendaciones
- [ ] Charla técnica o blog sobre PETs aplicadas
- [ ] CDPSE o ISO 27701 LI
