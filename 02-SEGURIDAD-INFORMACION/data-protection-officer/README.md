## 📋 Descripción

Rol **regulatorio-técnico** definido por GDPR (Art. 37-39) y replicado por leyes de privacidad locales. Garantiza que la organización cumpla con privacidad de datos personales: bases de tratamiento, derechos ARCOS, transferencias internacionales, brechas notificadas, evaluaciones de impacto (DPIA/PIA), registros (RoPA).

En Colombia el rol equivalente es **Oficial de Protección de Datos** bajo **Ley 1581 de 2012** + **Decreto 1377 de 2013**, con SIC (Superintendencia de Industria y Comercio) como autoridad.

DPO **debe ser independiente** (no puede ser el CISO ni el CIO en GDPR strict) — es un rol único en su naturaleza híbrida legal-técnica.

## 👤 Perfiles que llegan al DPO

- **Abogados con base técnica** — ruta más común
- **GRC / Compliance Officers** — privacy specialty
- **Privacy Engineers** — más técnicos (overlap con `privacy-engineer`)
- **Ex-auditores con foco privacy**
- **vDPO / Fractional DPO** — multi-cliente independiente

## 🗺️ Roadmap (12 meses)

### Fase 1 — Fundamentos (0-3 meses)
- [ ] GDPR completo leído (no solo summary)
- [ ] Ley 1581/2012 + Decreto 1377/2013 + circulares SIC
- [ ] LGPD Brasil + CCPA basics (cross-border)
- [ ] CIPP/E o CDPSE empezado

### Fase 2 — Aplicación (3-6 meses)
- [ ] RoPA (Records of Processing Activities) ficticio para empresa modelo
- [ ] DPIA completa: ejemplo procesamiento de alto riesgo
- [ ] DSAR workflow end-to-end
- [ ] **CIPP/E aprobado** + CIPM empezado

### Fase 3 — Profesional (6-12 meses)
- [ ] OneTrust o TrustArc operado en proyecto real / simulado
- [ ] Programa privacy pyme: políticas + RoPA + DPIA + training
- [ ] CIPM aprobado o CIPT
- [ ] Especializarse: AI privacy / health / finance / cross-border
